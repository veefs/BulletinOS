# TODO list / logistic planning & ideal features

## Feature List

### Physical Display

- Seamless display with case - Ideally the only cords we have is a power cable, and perhaps an ethernet cable if you don't want to connect it to wifi, but other than that it shouldn't jump out at you or look ugly or anything like that

- For the case in specific, it should have some sort of mounting point, as well as room for the microphone as well as the raspberry pi itself and we need a short HDMI cable in order to connect these two things

- For the size we already decided on a 14" x 8" display 14" tall 8" wide, but other options would also be very nice, I think the case and the hardware aspect is cool,but where I think this will really shine is the software aspect of the build

- For the audio output it also shouldn't be overly loud, obviously the volume will be configurable, but as a rule of thumb it shouldn't be "annoying"

### Dashboard / API features

- For the dashboard in specific I want the user to go through the following proces:

1. Pick a theme / general color theme we can have dark mode, light mode, catpuccin, etc, but also maybe have it so users can configure the UI with css

2. Set up their API key so that they can wirelessly populate the screen

3. Have AI agents or other things call the API key with some parameters maybe if we are json we can do like

```json

"Dashboard section", "TO-DO"
"Text", "You need to do your math homework"
```

Note that this is psuedo code but reflects the general idea of what I want to do
